# SOC Alert Investigations

This repository contains selected alert investigations completed during hands-on blue team training.

The goal of this project is to document practical work in:
- alert triage
- log analysis
- evidence correlation
- true positive vs false positive classification
- SOC-style analyst writing

A major part of this portfolio also supports my preparation for the SAL1 retake, with a strong focus on improving short-form analyst comments and structured case documentation.



## What this repository shows

- Investigation of alerts from low to critical severity
- Exposure to multiple categories such as malware, phishing, web attacks, persistence, and data leakage
- Practice in writing concise analyst comments using a 5Ws-style flow
- Structured reporting based on raw telemetry and supporting evidence



## Alert Categories

- Malware Execution
- Phishing / Initial Access
- Persistence
- Credential Access
- Lateral Movement
- Web Attacks
- Data Leakage / Exfiltration
- Miscellaneous



## Writing Formats Used

Each alert may be documented in three forms:

### 1. Analyst 5Ws Version
Short SOC-style analyst comment focused on:
- when
- who
- where
- what happened
- why it is suspicious

### 2. Full Structured Alert Report
Complete write-up using:
- Alert Name
- Alert Severity
- Alert Time
- Alert Description
- Context
- Evidence
- Analyst Comment

### 3. Portfolio Case Version
A cleaner public-facing version focused on:
- summary
- evidence
- analysis
- verdict
- response
- skills demonstrated
